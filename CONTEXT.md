# 🧠 CONTEXT: VDI AI Control System

## 1. Общая идея проекта

Мы разрабатываем систему:

```text
VDI AI Control Center
```

Это **персональный AI-ассистент**, который:

```text
- управляет удалённым компьютером (VDI)
- видит экран через видеопоток (OBS)
- управляет мышью и клавиатурой через ESP32 (HID)
- анализирует интерфейсы (через OCR + vision)
- выполняет задачи (через AI-агентов)
- хранит память (мультимодальную)
- взаимодействует с пользователем через web UI
```

---

## 2. Ограничения системы

Ключевое ограничение:

```text
VDI = полностью закрытая среда
```

То есть:

```text
❌ нет доступа к API
❌ нет доступа к DOM
❌ нет буфера обмена
❌ нет сети
```

Единственное взаимодействие:

```text
✔ экран (через OBS)
✔ мышь и клавиатура (через ESP32)
```

Это критично для архитектуры.

---

## 3. Как работает система

Цикл работы:

```text
1. Получаем кадр экрана VDI
2. Анализируем (OCR + Vision)
3. Понимаем, что на экране
4. Планируем действие
5. Показываем пользователю (если нужно)
6. Выполняем через ESP32 (клик/ввод)
7. Получаем новый кадр
8. Проверяем результат
9. Сохраняем всё в память
```

---

## 4. Основные компоненты системы

### 4.1 Web UI

Это основной интерфейс пользователя.

Он должен позволять:

```text
- видеть экран VDI
- кликать по экрану (как удалённый рабочий стол)
- вводить текст
- общаться с агентами
- видеть план действий
- подтверждать действия
- редактировать сообщения
- смотреть память
- смотреть историю
- управлять сценариями
```

---

### 4.2 Vision System

Обрабатывает экран:

```text
- OCR текста
- распознавание элементов UI
- определение зон (формы, таблицы, доски)
- сравнение экранов
```

---

### 4.3 ESP32 HID Bridge

Исполнитель:

```text
- эмулирует мышь
- эмулирует клавиатуру
- получает команды от backend
```

---

### 4.4 AI Agents

Система агентов:

```text
Supervisor Agent — главный координатор
VDI Agent — работает с экраном
Task Agent — задачи команды
Mail Agent — почта
Telegram Agent — чаты
Calendar Agent — встречи
Voice Agent — аудио/встречи
Message Writer — тексты
```

---

### 4.5 Memory System (очень важно)

Система памяти — ключевая часть.

Она хранит:

```text
- что система видела (экраны)
- что слышала (аудио)
- что читала (почта, чаты)
- что делала (действия)
- что решила (решения)
- что пользователь исправил
- как пользователь предпочитает работать
```

## 6. Approval System

Любое действие, которое меняет внешний мир:

```text
- отправка сообщения
- изменение задачи
- создание встречи
- сохранение формы
```

Должно проходить через:

```text
черновик → редактирование → подтверждение → выполнение
```
